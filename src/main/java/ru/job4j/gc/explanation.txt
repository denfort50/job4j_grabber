1. Я создал модель данных User, у которого 3 поля String, 1 поле int и 2 поля float.
2. 5 объектов User занимают 420 байт или 0,000401 мегабайт (String = 24 байта, int/float = 4 байта).
3. Объект состоит из заголовка, примитивных типов, ссылочных типов и выравнивания для кратности 8 байтам.
В 32-х разрядной системе размер заголовка — 8 байт. Это и будет размер пустого объекта без полей.
4. Установил ключи -Xms2m -Xmx4m. При запуске main увидел, что до создания объектов было меньше свободной памяти,
чем после, следовательно, сборщик мусора сделал свою работу.

UPD: подробный расчет на примере объекта user1.
1) Объект String состоит из:
        заголовка (8 байт),
        private final int offset (4 байта);
        private final int count (4 байта);
        private int hash (4 байта);
        private final char value[] (ссылка 4 байта + сам объект массива:
                                        заголовок 8 байт + 4 байта на длину массива
                                        + примитивы char по 2 байта каждый
                                        + выравнивание для кратности 8).

   Таким образом,
   "Alexandr" (24 байта + 12 + [2 * 8] + 4 = 56 байт),
   "Ivanov" (24 байта + 12 + [2 * 6] + 0 = 48 байт),
   "Blockchain Developer" (24 байта + 12 + [2 * 20] + 4 = 80 байт)

2) int всегда занимает 4 байта

3) float всегда занимает 4 байта

Итого user1 = 56 + 48 + 80 + 4 + 4 + 4 = 196 байт
